How should the Timestamp column display time?

This affects how users read the entries table. Pick the format that makes most sense for debugging.

A
Absolute clock time Matches Google
Shows the actual wall-clock time each request fired (HH:MM:SS.mmm). Lets you correlate with server logs, browser console, or other tools.
TimestampStatusURLTime
06:17:56.461302/nof-hdcc1589ms
06:17:57.794303/oauth2/v1/authorize300ms
06:17:58.094302/fed/v1/use533ms
06:18:00.440200/servererror.jsp266ms
06:18:00.722200/css/login_page.css25ms
B
Relative from first request Performance focus
Shows elapsed time from the very first request in the HAR (T+0ms, T+1333ms…). Great for measuring page load stages, not great for correlating with external logs.
TimestampStatusURLTime
+0ms302/nof-hdcc1589ms
+1333ms303/oauth2/v1/authorize300ms
+1633ms302/fed/v1/use533ms
+3979ms200/servererror.jsp266ms
+4261ms200/css/login_page.css25ms
C
Both with a toggle Most flexible
Column header click cycles between absolute and relative. Slightly more complex to build, but gives users the best of both worlds.
⟳ TimestampStatusURLTime
06:17:56.461302/nof-hdcc1589ms
06:17:57.794303/oauth2/v1/authorize300ms
+0ms → +1333ms302/fed/v1/use533ms
(toggles on click)200/servererror.jsp266ms
Click the column header to toggle between formats
